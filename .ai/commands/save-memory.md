# /save-memory

把当前对话里值得长期记住的**业务**认知，写入 `.ai/memory/MEMORY.md`。

## 默认加载 vs 写入位置

- **默认只加载** `memory/MEMORY.md`
- **业务永久记忆** → 在本文件追加/更新 `## 标题` 小节
- **计划与待拍板** → `decisions/current.md`（用 `/save-decisions` 归档）
- **过期业务小节** → `archive/memory/<标题>.md`（从 MEMORY 移出整节）

## 触发方式

- `/save-memory` — 扫描最近对话，挑 1–3 条候选给用户确认
- `/save-memory <自由文本>` — 强制候选，跳过扫描

## 执行流程

### 模式 A：无参数

1. 回看约 10 轮对话，挑**业务相关**候选（产品、领域、流程、对外约定）
2. 黑名单：代码路径、git、单次 bug、任务进度、`rules/` / `docs/` 已有内容、纯技术实现细节
3. 报告候选：章节标题、正文预览（`Context` / `Rule`）
4. `AskUserQuestion` 逐条：保存 / 跳过 / 编辑
5. 确认后落盘：在 `MEMORY.md` 的 `---` 分隔线下方追加或合并同名 `##` 小节

### 模式 B：带自由文本

用户文本视为已通过初筛，建议章节标题，确认后落盘。

## 分类信号（写入同一文件，用章节标题区分）

| 信号 | 章节标题示例 |
|------|----------------|
| 产品/功能边界 | `## 订单状态机` |
| 领域术语与口径 | `## 用户等级定义` |
| 客户或项目约束 | `## XX 客户部署限制` |
| 已确认的需求取舍 | `## 列表页默认不分页` |

跨会话**技术**裁决未进 `rules/` 时 → `decisions/current.md`，不要用本命令。

## 归档过期小节

用户确认某 `##` 小节已过时：

1. 将该节全文写入 `archive/memory/<标题>.md`（标题 slug，重名加 `YYYY-MM-DD-` 前缀）
2. 从 `MEMORY.md` 删除该节

## 小节格式（强制）

```markdown
## 标题

**Context:** …

**Rule:** …
```

每小节 ≤ 15 行。

## 不允许

- 把编码规范、组件 props 写入 `MEMORY.md`
- 预加载 `decisions/`、`docs/`（除非任务相关）
- 在 `memory/` 下新建除 `MEMORY.md` 外的记忆文件
- 不在用户确认前落盘
